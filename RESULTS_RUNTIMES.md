# Сравнение рантаймов: Django vs Dart vs Go vs Rust vs Erlang (2026-06-21)

Одна и та же API-реализация Task Studio, переписанная на разных стеках, прогнана
**идентичным** нагрузочным свипом на **одном и том же железе**, чтобы сравнить
реальную производительность рантаймов (а не «слабый ли сервер»).

## Стенд и метод

- **Сервер:** RackNerd VPS, **$20/год**, ~4 vCPU (shared/oversold), 2 ГБ RAM,
  Debian 13, Postgres 18. Все стенды — на этой же машине.
- **Клиент:** Windows (Xeon E5-2678 v3), `bench_rest.py` (closed-loop, нулевая
  пауза), с локальной машины через интернет (US ↔ другой конец планеты).
- **Доступ:** каждый стенд за nginx+TLS на `https://tasks.titov.dev/<name>bench`
  (тот же сертификат, отдельный path), изолированная БД на стенд. Прод-Django не
  трогался во время прогонов (кроме своего собственного замера).
- **Метод:** свип concurrency, `--duration 10`, warmup 2с. На Windows
  `PYTHONUTF8=1`.

> ⚠️ **Latency здесь ≈ сетевой RTT, а НЕ время сервера.** Сервер в США, клиент на
> другом конце планеты → базовые ~155–185мс p50 — это межконтинентальный
> round-trip. Серверное время — единицы-десятки мс. **Значимая метрика —
> throughput (RPS), ошибки и форма кривой**, а не абсолютная latency.

> ⚠️ **concurrency ≠ юзеры.** c=100 closed-loop ≈ тысячи реально активных
> пользователей.

### Параллелизм (все на 4-ядерном боксе)

| Рантайм | Модель | Воркеры |
|---|---|---|
| Django | gunicorn, sync-воркеры | ~8 процессов |
| Dart×1 | один isolate (event loop) | 1 ядро |
| Dart×4 | multi-isolate, общий сокет | 4 изолята |
| Go | net/http + goroutines | GOMAXPROCS=4 |
| Rust | axum + tokio multi-thread | 4 worker threads |
| Erlang | Cowboy + BEAM (preemptive), WS-хаб на ETS | 4 schedulers |

Хеш паролей: Django/Dart/Rust — PBKDF2-SHA256 (формат Django); Go — bcrypt;
Erlang — PBKDF2 чистым Эрлангом (на стенде итерации снижены, см. ниже). Хеш не на
горячем пути бенча (только signup в setup), на измеряемые sync/heartbeat не влияет.

## Выводы

1. **Heartbeat (чистая конкуренция, без БД) — чемпион Эрланг.** 612 RPS на c=100
   (выше всех), и — главное — **держит 606 на c=200, где Go проседает до 303, а
   Rust до 383**. Это суперсила BEAM: тьма дешёвых процессов + преемптивный
   шедулер не дают одному медленному запросу задушить остальные, p50 железно
   158мс. На лёгком пути с высокой конкуренцией «обсирается» как раз Go/Rust
   (валятся вдвое), а Эрланг — нет.

2. **Sync (DB-bound) — Go и Rust co-чемпионы, Эрланг чуть позади на верху.** До
   c=40 все трое вровень (~225). Дальше Go (415) и Rust (393–409) уходят, Эрланг
   садится на ~286–340 — в полосе между Dart×4 и Go. Причина не язык, а паттерн
   доступа к БД: `sync/download` делает ~10 последовательных checkout'ов из пула
   poolboy (round-trip к воркеру каждый), тогда как Go/Rust бьют в пул напрямую.
   Оптимизируется одним коннектом на запрос (не делалось).

3. **Грабля poolboy (важно!).** Первый прогон Эрланга коллапсировал на c=40 (19с,
   2 RPS). Это была НЕ вина рантайма, а конфиг пула: `max_overflow>0` заставляет
   poolboy создавать коннекты **внутри своего gen_server**, и блокирующий
   `epgsql:connect` там морозит весь пул + гонит connect/disconnect churn. Фикс —
   фиксированный пул без overflow (одна строка). После — ровно с Go.

4. **Go/Rust ~6× над Django на sync, упёрлись в Postgres/железо** (~400 RPS) — не
   в язык. Эрланг туда же дотягивается на c=40.

5. **Django** worker-bound: плато sync ~65, heartbeat ~160. Любой
   async/многоядерный рантайм снимает лимит без добавления памяти.

6. **Хвосты p99 = 7–13с** на c≥160/200 у всех — установка TLS через океан, не
   сервер.

### Итог для этой машины

| | Throughput | p50 | Footprint | Цена разработки |
|---|---|---|---|---|
| **Go** | 🥇 sync, 🥈 heartbeat | 🥇 плоский | 🥇 крошечный бинарь | 🥇 низкая |
| **Rust** | 🥇 sync, 🥈 heartbeat | 🥇 плоский | 🥇 крошечный бинарь, без GC | высокая |
| **Erlang** | 🥇 **heartbeat**, sync чуть ниже | 🥇 плоский | средний (BEAM рантайм) | 🔴 грабли/экспертиза |
| **Dart×4** | середина | средний | средний (N изолятов × heap) | низкая |
| **Django** | 🔴 | растёт | 🔴 самый тяжёлый | низкая |

- **На горячем пути все четыре переписки (Go/Rust/Erlang/Dart×4) кратно бьют
  Django** и упираются в Postgres/железо, а не в язык.
- **Go — прагматичный чемпион:** топ-перф (sync 🥇, heartbeat близко) при самой
  низкой цене разработки и тривиальном деплое (cross-compile → один бинарь).
- **Rust = Go по перфу**, дороже в разработке.
- **Erlang** уникален на **массовой конкуренции/realtime** (heartbeat 🥇, держит
  c=200) — это его ниша (WS, presence, много живых коннектов). На DB-bound CRUD
  чуть позади и требует знания потрохов (poolboy, PBKDF2-в-NIF). На его родной БД
  (Mnesia/ETS, без хопа в Postgres) sync бы тоже выстрелил — но это уже сравнение
  СУБД, а не рантаймов.
- **Прагматически:** Django адекватен реальному трафику (см. `RESULTS.md`).
  Переписывание оправдано упором в ресурсы/footprint, а не «цифры красивее».

### Команды (для воспроизведения)

```bash
# каждый стенд: https://tasks.titov.dev/<name>bench  (gobench/dartbench/rustbench/erlangbench)
PYTHONUTF8=1 python bench_rest.py --target https://tasks.titov.dev/erlangbench \
  --scenario sync --concurrency $c --duration 10        # c = 1,5,10,20,40,80,160
PYTHONUTF8=1 python bench_rest.py --target https://tasks.titov.dev/erlangbench \
  --scenario heartbeat --concurrency $c --duration 10   # c = 10,50,100,200
# Django baseline: --target prod  (см. RESULTS.md)
```
